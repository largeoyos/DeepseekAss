## 一、快速开始

### 1.1 环境要求

- Python 3.9+
- 有效的 DeepSeek API Key

### 1.2 安装与启动

```bash
pip install -r requirements.txt
python gui_main.py
```

### 1.3 首次登录

启动后弹出登录对话框：

- **已有账号**：输入用户名和密码登录
- **注册新账号**：勾选"注册新账号"，输入用户名和两次密码确认
  - 密码要求：至少 6 位，需包含字母和数字
  - 密码用于加密所有本地数据，请妥善保管

首次登录会自动检测 API Key：
- 如果在配置文件中（`.env`）设置了 `DEEPSEEK_API_KEY` 则自动使用
- 否则弹出输入框，手动输入 API Key 和 Base URL（可选，默认 `https://api.deepseek.com`）
- 输入后自动验证，失败会提示

### 1.4 迁移旧数据

首次登录时，如果检测到旧版（未加密）的用户数据目录，系统会自动提示迁移。迁移会将旧数据加密复制到当前用户目录，仅缺失的数据会被迁移，不会覆盖新数据。

## 二、界面概览

窗口采用三栏式布局：左侧导航栏、中间控制面板、右侧对话区。

### 2.1 左侧导航栏（76px 宽）

顶部显示 DeepSeek 品牌图标，下方三个模式切换按钮：

| 按钮 | 模式 |
|------|------|
| 角色扮演 | 角色扮演模式 |
| 小说写作 | 小说写作模式 |
| 续写 | 续写小说模式 |

底部固定按钮：Token 日志、设置。

### 2.2 顶部工具栏

显示当前状态：模式、模型、当前书籍、状态（就绪/生成中）。

### 2.3 左侧控制面板（约 450px，可滚动）

通用控制区域（所有模式下均显示）：

| 控件 | 说明 |
|------|------|
| 模型选择 | 下拉框选择模型 |
| 预设方案 | 参数预设，可选保守/中庸/狂野/自定义 |
| 温度 | 滑块 0.00~2.00，控制创造力 |
| top_p | 滑块 0.00~1.00，采样阈值 |
| 频率惩罚 | 滑块 -2.00~+2.00，越高越不重复 |
| max_tokens | 数值框 1~300000，单次最大输出 |

预设方案默认值：

| 预设 | 温度 | Top P | 频率惩罚 | 适用场景 |
|------|------|-------|---------|---------|
| 保守 | 0.30 | 0.85 | +0.30 | 事实性内容、严格续写 |
| 中庸 | 0.70 | 0.90 | 0.00 | 日常聊天、常规写作 |
| 狂野 | 1.20 | 0.95 | -0.20 | 创意思维、发散情节 |

自定义参数会自动保存到用户预设中，可在设置中管理。

### 2.4 右侧对话区

- **上方**：QWebEngineView Markdown 渲染聊天记录，支持深色/浅色主题
- **下方**：输入框（Enter 换行，Ctrl+Enter 发送）+ 发送/停止按钮

生成过程中输入框旁显示红色停止按钮，点击立即终止生成，已输出内容保留。

## 三、三种工作模式

| 模式 | 策略类 | 用途 |
|------|--------|------|
| 角色扮演 | RolePlayStrategy | 让 AI 扮演角色与你对话 |

## 四、角色扮演模式

### 4.1 基础设置

| 控件 | 说明 |
|------|------|
| 角色列表（多选） | 从角色簿中选择参与对话的角色 |
| 回复方式 | 角色回答（第一人称）/ 旁白描述（第三人称） |
| 应用设定 | 将选中的角色和回复方式写入 AI 指令 |

### 4.2 角色簿系统（Character Book）

角色簿是角色扮演模式专用的角色管理系统，与世界书相互独立。

**角色档案字段：**
- 姓名、别名
- 身份（如"王国的暗夜刺客"）
- 外貌、性格、说话风格
- 背景故事、目标、禁忌、备注

**自动记忆系统：**
每轮对话后，AI 会自动提取以下信息并保存到该角色的记忆中：
- 经历的事情
- 当前状态与情绪
- 与其他角色的关系变化
- 最近的行为
- 关键对话

**管理入口：**
- **新建角色**：打开角色档案编辑对话框
- **编辑角色**：修改当前选中角色的档案
- **角色簿**：打开角色簿总览面板

### 4.3 对话管理

| 操作 | 说明 |
|------|------|
| 新建私聊 | 创建与单个角色的对话 |
| 新建群聊 | 创建与多个角色的群组对话 |
| 时间线 | 查看当前会话中发生的事件时间线 |
| 保存对话 | 保存当前对话（含角色、回复模式等信息） |
| 加载对话 | 加载历史对话，自动恢复当时的模式、模型和角色设定 |

### 4.4 特殊语法

OOC（Out of Character，跳出角色）提问请使用 `「」` 或 `【】` 包裹，AI 会以 AI 身份回复而非角色身份。

### 5.1 书架管理

| 操作 | 说明 |
|------|------|
| 书架下拉框 | 选择/切换已有小说 |
| 新建 | 创建新小说，输入书名后自动创建书架目录 |

| 风格基调 | 7 种风格基调可选 |

**题材列表与默认参数：**

| 题材 | 推荐温度 | 频率惩罚 |
|------|---------|---------|
| 玄幻/仙侠 | 0.85 | +0.40 |
| 西幻 | 0.85 | +0.40 |
| 科幻 | 0.75 | +0.50 |
| 历史 | 0.70 | +0.60 |
| 都市 | 0.80 | +0.50 |
| 悬疑/惊悚 | 0.70 | +0.30 |
| 武侠 | 0.80 | +0.40 |
| 言情 | 0.90 | +0.30 |
| 末日废土 | 0.80 | +0.40 |
| 恐怖 | 0.75 | +0.20 |
| 轻小说 | 0.90 | +0.30 |
| 情色 | 0.90 | +0.20 |
| 不限风格 | — | — |

### 5.3 创作设定面板

| 控件 | 说明 |
|------|------|
| 主角设定 | 主角的背景、性格、外貌、能力等 |
| 世界观/背景故事 | 时代背景、核心设定、势力格局等 |
| 写作要求 | 对本章写作风格、内容、节奏的要求 |
| 作者计划 | 未来的写作意图（故事主线目标、阶段任务、角色弧光），标明"未来意图，非已发生事实" |
| 全局偏好提示词 | 持续生效的个性化写作偏好，所有生成环节自动注入 |
| 本章情节（可选） | 你希望本章发生的关键情节和场景 |
| 目标字数 | 本章目标字数，范围 100~100000，步长 500，默认 40000 |

主角设定、世界观、写作要求等文本区域修改后自动保存。也可使用「保存设定/加载设定」按钮手动操作。

### 5.4 章节生成流程

点击「生成下一章」后，系统自动执行：

1. **收集上下文**：主角设定 + 世界观 + 写作要求 + 作者计划 + 全局偏好 + 本章情节
2. **加载前情提要**：短篇小说（<=5 章）读取完整摘要；长篇小说自动压缩早期章节为约 300 字概览，保留最近章节详细信息
3. **构建连续性契约**：从上一章结尾点、角色当前状态、活跃剧情线、未解伏笔、待完成剧情中构建约束
4. **加载生成历史**：读取历史生成记录，避免 AI 重复已用过的思路
5. **加载世界书**：已提取的角色、地点、规则、剧情线、伏笔
6. **AI 生成章节正文**
7. **自动保存**：章节文件 + 生成记录（含提示词、参数等完整信息）
8. **追加剧情摘要**：提炼当前章节摘要，合并到整体剧情摘要
9. **自动补写**：字数不足目标 80% 时，自动调用 AI 扩充章节（保留原有情节，丰富细节和对话）
10. **更新世界书**：提取本章新角色、新地点、新规则、新伏笔，合并到世界书

### 5.5 章节版本管理

每章可保留多个版本，互不覆盖。

| 操作 | 说明 |
|------|------|
| 新建版本 | 编辑后保存为新版本 |
| 切换活跃版本 | 指定某版本为"活跃"版本，只有活跃版参与剧情摘要 |
| 删除版本 | 删除某版本（不能删除活跃版本） |

版本文件命名格式：`第N章_标题_v版本号.txt`

### 5.6 章节编辑器

在章节树中双击节点打开编辑器：

- 左侧文件列表显示当前章节所有版本
- 右侧编辑区支持修改内容
- 每 30 秒自动保存草稿到 `.drafts/` 目录
- 保存时可选择"另存为新版本"和"设为活跃版本"
- 显示中文字数统计

### 5.7 辅助功能

| 功能 | 说明 |
|------|------|
| 世界书 | 查看/编辑 AI 自动提取的世界书数据 |
| 章节管理 | 打开章节树管理界面 |
| 批量导入章节 | 选择包含编号 .txt 文件的文件夹，批量导入为小说章节，自动提取世界书 |

## 六、续写小说模式

用于为已有的 .txt/.md 文章续写后续章节。

### 6.1 选择源文档

| 控件 | 说明 |
|------|------|
| 源文档 | 选择一个 .txt/.md 文件 |
| 或指定文件夹 | 选择一个文件夹，读取其中所有 .txt/.md 文件（按文件名排序） |

### 6.2 分析与导入

点击「分析源文档并导入设定」后，AI 自动执行：

1. **语义分段**：按话题转折自动划分逻辑段落
2. **世界提取**：逐段提取角色、地点、规则、事件、剧情线
3. **跨段合成**：3 段以上时自动合成去重
4. **生成小说设定**：从提取的数据生成世界观背景、主角设定、写作要求、作者计划
5. **创建书架**：自动创建小说项目，保存世界书和设定

完成后弹出分析结果总览窗口，包含生成的设定、角色、地点、规则、剧情线、事件等选项卡。

### 6.3 快速续写

点击「直接续写」跳过分析流程，将源文档全文（或部分）直接作为上下文续写。

### 6.4 续写操作

| 控件 | 说明 |
|------|------|
| 续写要求 | 对续写风格、视角、节奏的要求 |
| 续写剧情（可选） | 填写具体剧情走向和关键事件 |
| 目标字数 | 目标字数，默认 10000 |

三个生成入口：

| 按钮 | 说明 |
|------|------|
| AI 建议发展方向 | AI 根据已分析的设定和活跃剧情线，给出 3~5 个发展方向建议，选择后自动续写 |
| 我指定剧情 | 将"续写剧情"框的内容传给 AI 续写 |
| 生成下一章 | 直接续写，自动注入前文剧情和设定 |

### 6.5 分段预览

选择源文档后弹出分段预览对话框：

- 左侧：段落列表（含标题预览）
- 右侧：当前段落完整内容
- AI 重新分段：不满意自动划分时手动触发
- 文件夹模式下可勾选需要包含的文件

## 七、世界书系统

世界书（World Bible）是小说设定和剧情进展的结构化知识库，由 AI 在每次生成章节后自动提取和更新，也可手动编辑。

### 7.1 数据类别

| 类别 | 内容 |
|------|------|
| 角色 | 姓名、别名、特质、关系、状态（存活/死亡/失踪/变身）、重要性（主要/次要/龙套）、关键细节、对话、动机、弧光、位置、目标、情绪、最近行为、知识边界、未解冲突 |
| 地点 | 名称、描述、意义、关键细节、氛围 |
| 规则 | 世界规则和设定约束 |
| 时间线 | 按章节记录的重大事件及其意义、伏笔线索 |
| 剧情线 | 状态（活跃/已解决/休眠）、涉及角色、描述、预期收束方式、伏笔关联 |
| 设定与伏笔 | 关键世界观设定、全局伏笔、关键对话 |

### 7.2 自动提取机制

每生成一章，AI 自动提取新内容（前 40000 字符）并合并到世界书：
- 同名/别名自动合并（如"小张"和"张伟"会被识别为同一人）
- 重要性排序（主要角色排前）
- 冲突检测（如已标记死亡的角色再次出现）

### 7.3 世界书对话框

通过小说或续写面板的「世界书」按钮打开，包含以下功能：

**九大选项卡：**
- 角色、地点、规则、时间线、剧情线、设定、伏笔、关键对话、冲突警告

**工具栏操作：**
| 功能 | 说明 |
|------|------|
| 合并角色 | AI 检测并合并重复的角色条目 |
| 标记已解决 | 按关键词匹配标记待处理项为已解决 |
| 锁定核心设定 | 将关键设定锁定，不会被自动隐藏 |
| 隐藏低优先级 | 自动隐藏次要/龙套角色等低重要性条目 |
| 按章节查看 | 按来源章节筛选条目 |
| 添加伏笔 | 填写伏笔内容和预期收束方式 |

**卡片视图：** 每条数据以卡片形式展示，含标签、元数据和可展开详情。

**筛选栏：** 按类型、状态（全部/活跃路径/可见/隐藏/已解决）、来源章节筛选。

**保存时自动触发一致性审计。**

### 7.4 一致性审计

每次保存世界书时自动运行审计，检测：

| 检查项 | 说明 |
|--------|------|
| 空类别 | 是否有某类数据完全缺失 |
| 缺失来源 | 角色/地点等是否缺少来源章节 |
| 角色名冲突 | 不同角色是否使用了相似名称 |
| 死亡状态冲突 | 已标记死亡的角色是否仍有活跃状态 |
| 关系噪音 | 关系描述是否合理 |
| 地点重复 | 不同名称是否指向同一地点 |
| 时间线倒序 | 事件顺序是否合理 |
| 休眠剧情线 | 8 章以上未推进的剧情线 |
| 未解伏笔 | 8 章以上未推进且缺少下一步计划的伏笔 |
| 规则重复 | 世界规则是否有重复 |

### 7.5 Prompt 中注入形式

世界书支持两种注入策略：

- **简单模式**：按重要性排序，截取到最大条目数
- **相关度评分**：根据当前章节内容语义评分，按分数 + 重要性 + 新鲜度排序，分三级注入（核心设定恒定注入、本章相关内容、低优先级存档）

## 八、章节树管理

章节树是一个图形化的章节结构管理工具，以树状图展示各章节之间的关系。

### 8.1 打开方式

在小说写作或续写面板中点击「章节管理」按钮。

### 8.2 视图操作

| 操作 | 说明 |
|------|------|
| 缩放 | + / - / 100% / 自适应 按钮，也可 Ctrl+滚轮缩放 |
| 拖拽 | 拖拽画布查看不同区域 |
| 节点点击 | 点击章节节点，右侧显示详情：章节号、标题、版本、父节点、子节点、创建时间、字数、摘要、生成参数、提示词、内容预览 |

### 8.3 节点操作

右键或选中节点后可使用以下操作：

| 操作 | 说明 |
|------|------|
| 切换分支 | 切换到选中的分支路径（可选重建记忆和世界书） |
| 编辑内容 | 打开章节编辑器 |
| 润色 | 调用 AI 润色当前章节，生成新版本 |
| 重写 | 调用 AI 基于原始需求重新创作，生成新版本 |
| 导出章节 | 导出当前章节 |
| 插入中间章 | 在选中章节后插入新章 |
| 删除子树 | 删除选中节点及其所有子节点 |
| 强制重新提取世界书 | 从章节正文重新提取世界书数据 |

**分支切换**会改变当前写作路径。如果你之前写了分支 A 的 5 章，现在想切换到分支 B 继续写，可以通过此操作完成。

## 九、连续性审计与修复

在每次生成章节时，系统可以自动进行连续性检查。

### 9.1 连续性审计

审计以下方面的一致性：

- 角色位置、时间线、行为连续性
- 动机、情绪、关系递进是否合理
- 角色知识边界（角色不应知道 TA 不该知道的事）
- 世界观规则一致性
- 伏笔管理（是否被提前解决、被遗忘、解决方式错误）
- 章节契约满足度（上一章立下的剧情承诺是否兑现）

### 9.2 连续性修复

基于审计结果，AI 会以最小改动原则重写冲突段落，保持原风格和叙事流畅性。

## 十、导出功能

### 10.1 单章导出

### 10.2 全书导出

## 十三、数据安全与存储结构

### 13.1 加密机制

- 使用 **Fernet 加密**（对称加密）
- 加密密钥通过 **PBKDF2-HMAC-SHA256**（600,000 次迭代）从登录密码派生
- 64 字节派生输出：前 32 字节用于身份验证哈希，后 32 字节用作加密密钥
- 修改密码时自动用新密钥重新加密全部用户数据
- 加密文件以 `.enc` 后缀标识

### 13.2 存储结构

```
users/
└── <UUID>/
    ├── config.enc                     # API Key / Base URL
    ├── user_prefs.enc                 # 全局偏好提示词
    ├── character_book.json.enc        # 角色簿（角色扮演）
    ├── bookshelf/
    │   └── <小说名>/
    │       ├── meta.json.enc          # 元信息（设定、章节版本）
    │       ├── plot_summary.txt.enc   # 剧情摘要
    │       ├── world_bible.json.enc   # 世界书
    │       ├── 第1章_标题_v1.txt.enc  # 章节正文
    │       └── .generation_history/   # 生成记录日志
    ├── conversations/
    │   └── <对话名>.json.enc          # 对话历史
    └── token_log.json.enc             # Token 使用日志
```

### 13.3 安全须知

- **密码丢失无法恢复数据**，加密密钥基于密码派生，没有密钥即无法解密
- API Key 和 Base URL 以加密形式存储
- 设置高强度密码保护所有本地数据

## 十四、常见问题

**Q：忘记密码了怎么办？**
A：加密密钥由密码派生，忘记密码无法恢复数据。请务必牢记密码。

**Q：如何更换 API Key？**
A：登录后点击左侧导航栏设置图标 → API Key 选项卡 → 修改并验证。

**Q：生成速度慢怎么办？**
A：选择 `deepseek-v4-flash` 模型可显著提速；降低 `max_tokens` 减少等待时间；目标字数不要设置过高。

**Q：生成被中断了，已生成的内容还在吗？**
A：点击停止按钮后，已流式输出的内容保留在对话区，但尚未保存到书架，需要手动复制保存。

**Q：重新生成会覆盖已有章节吗？**
A：不会。润色或重写都会保存为新版本（v+1），原有版本保留，可通过章节管理切换活跃版本。

**Q：角色簿和世界书有什么区别？**
A：角色簿（Character Book）专用于角色扮演模式，管理角色档案和对话记忆；世界书（World Bible）专用于小说写作模式，管理整个故事的世界观、角色、地点、剧情线等。

**Q：切换分支后之前写的内容会丢失吗？**
A：不会。所有分支的内容都保存在书架中，切换分支时可以选择是否重建剧情摘要和世界书，原分支数据保留不变。

**Q：章节字数总是不够怎么办？**
A：系统会在字数不足目标 80% 时自动补写。也可以手动调高目标字数，或在写作要求中写明"详细描写场景和人物心理"。

**Q：我可以在多台电脑上使用吗？**
A：可以，但数据不自动同步。可以在设置中导出用户数据为 ZIP，在另一台电脑上导入。注意使用相同的密码。
