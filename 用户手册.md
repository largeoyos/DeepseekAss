# DeepseekAss 用户手册

本文档对应当前版本的 DeepseekAss 桌面客户端。界面或模型供应商发生变化时，以程序中的按钮、提示和设置项为准。

## 1. 开始使用

### 1.1 安装

建议使用 Python 3.10 或更高版本：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python gui_main.py
```

### 1.2 注册和登录

首次运行会打开本地账号窗口。

- 新用户点击“没有账号？去注册”。
- 用户名不能为空。
- 密码至少 6 位，并同时包含字母和数字。
- 注册完成后数据将绑定到该账号的加密密钥。
- 已有账号直接输入用户名和密码登录。

密码用于派生本地加密密钥，程序不保存可恢复的明文密码。密码遗失后无法解密原数据。

### 1.3 配置 API

登录后打开“设置中心 → API 与模型”。文字 API 是聊天、小说、续写、摘要、世界书和 Agent 的必要配置。

需要填写：

- 调用地址：OpenAI 兼容 API 的 Base URL；
- API Key；
- 模型名称。

点击“测试连接”确认配置。图片 API 是独立可选配置，不影响文字功能启动。

### 1.4 主界面

左侧窄导航栏提供四个工作区：

| 工作区 | 用途 |
|---|---|
| 角色扮演 | 角色私聊、群聊、人物书、时间线与场景控制 |
| 小说写作 | 从零创作、章节生成、Agent 顾问、章节树与世界书 |
| 续写小说 | 导入已有正文、分析设定、批量建书并继续写作 |
| Markdown 笔记 | 本地笔记、实时预览和导出 |

导航栏下方还有 Token 日志和设置中心入口。

主工作区通常由三部分组成：

- 左侧控制面板：模式参数和项目操作；
- 右侧内容区：对话、生成进度或 Markdown 渲染；
- 顶部状态区：当前模式、模型、运行任务和项目状态。

## 2. 模型和生成参数

### 2.1 模型选择

模型列表由以下来源合并：

- API 配置中的当前模型；
- 内置模型名；
- 用户收藏和自定义模型。

模型名必须与服务端实际支持的名称一致。

### 2.2 参数

| 参数 | 范围 | 作用 |
|---|---:|---|
| 温度 | 0.00～2.00 | 越高越发散，越低越稳定 |
| top_p | 0.00～1.00 | 控制候选词采样范围 |
| frequency penalty | -2.00～2.00 | 正值降低重复，负值提高复用倾向 |
| max_tokens | 1～300000 | 单次模型输出上限，仍受服务端限制 |

可使用参数预设，也可保存自定义预设。切换题材或任务时，应同时关注目标字数和模型输出上限。

### 2.3 全局写作偏好

全局偏好用于表达长期写作习惯，例如视角、描写密度、对白风格和禁用表达。它可能被注入章节生成、摘要、世界书、续写建议和 Agent 任务。

不要在全局偏好中写只对某一章有效的剧情；单章要求应填写在本章情节或写作要求中。

## 3. 角色扮演

### 3.1 角色档案和人物书

角色档案保存稳定信息，例如：

- 名称、身份和人物描述；
- 说话风格、目标和关系；
- 补充设定；
- 对话过程中积累的经历、知识状态和记忆。

角色簿与小说世界书相互独立：角色簿服务于角色扮演会话，世界书服务于小说项目。

### 3.2 创建会话

1. 在角色列表中选择角色。
2. 单角色选择“新建私聊”；多角色选择“新建多角色聊天”。
3. 选择发送者身份、回复角色和旁白策略。
4. 在输入框中发送消息。

会话可记录参与者、场景状态、发言策略、分支和时间线事件。

### 3.3 控制中心

控制中心用于查看或调整：

- 当前场景和在场角色；
- 必须回复的角色；
- 旁白是否启用；
- 玩家身份和发送者档案；
- 对话分支、消息操作和记忆变更。

对角色知识状态的修改应基于对话中已经发生的信息，避免让角色提前知道未公开事实。

### 3.4 保存、加载和导出

当前对话可以保存为独立记录。加载时会恢复可用的策略、角色、模型和会话状态。

支持导出 TXT、Markdown、HTML 和 DOCX。导出文件不再加密，请自行选择安全位置。

## 4. 小说写作基础

### 4.1 书架

在小说写作工作区可以：

- 新建书籍；
- 切换现有书籍；
- 重命名书籍；
- 删除书籍；
- 打开章节树、世界书、上下文策略和项目版本。

删除整本书属于破坏性操作。重要项目应先导出或创建项目快照。

### 4.2 小说设定

每本书可保存：

| 字段 | 含义 |
|---|---|
| 主角设定 | 身份、性格、能力、经历和核心目标 |
| 世界背景 | 时代、地点、社会结构、力量体系和核心规则 |
| 写作要求 | 文风、视角、节奏、限制和输出要求 |
| 作者规划 | 尚未发生的未来意图、阶段目标和角色弧线 |
| 题材/风格 | 为生成提供额外的类型和语气指导 |
| XP 模式 | 调整关系张力和成人向创作关注点，仅用于虚构创作 |

作者规划是未来意图，不应当被摘要或世界书当成正文已经发生的事实。

### 4.3 章节输入

生成前填写：

- 章节标题；
- 本章情节或希望发生的关键场景；
- 写作要求；
- 目标字数，范围 100～100000 字。

目标字数是模型约束，不是精确保证。超长章节更容易受到模型上下文和输出限制，建议按场景拆章。

## 5. 经典写作模式

在“设置中心 → Agent → 小说写作模式”选择经典模式。

经典模式生成时主要考虑：

- 小说主角设定、世界背景和写作要求；
- 作者规划和本章情节；
- 活跃路径的前情摘要；
- 当前角色状态、剧情线和伏笔连续性契约；
- 世界书中与本章相关的条目；
- 全局用户偏好和题材风格。

流程为：生成正文 → 连续性检查/修补 → 保存章节版本 → 生成摘要 → 更新世界书 → 创建项目快照。

如果摘要或世界书更新失败，已成功保存的章节不会因此删除。

## 6. Agent 写作模式

### 6.1 启用

打开“设置中心 → Agent”，选择“Agent 写作模式”。经典模式和 Agent 模式共享同一书架、章节树和世界书，可随时切换，但运行中的任务必须先结束。

可同时配置：

- 是否启用内置及书籍级 Skills；
- Agent 运行时：内置运行时或 LangGraph；
- 检索后端：基础关键词检索或 LlamaIndex 混合检索；
- Embedding 服务；
- 受控网页搜索。

可选后端配置无效时，程序可能回退到内置能力，并在状态信息中说明原因。

### 6.2 生成下一章

Agent 生成不是直接把整本书塞给模型，而是两阶段筛选。

规划阶段读取：

- 主角设定、世界背景、写作要求和作者规划；
- 本章标题、情节、目标字数和全局偏好；
- 连续性契约；
- 世界书轻量索引；
- 活跃路径章节摘要索引；
- 与任务匹配的 Skills。

规划模型输出：

- 章节目标；
- 场景目的、冲突和结果；
- 角色起始/结束状态；
- 需要推进的剧情线；
- 伏笔埋设、推进或回收动作；
- 本章需要展开的世界书实体；
- 需要读取的历史章节；
- 硬约束和规划备注。

上下文组装阶段会加入：

- 最近章节摘要；
- Agent 选中的历史摘要，且默认保留最近三章用于承接；
- 常驻、自动命中和手动选择的世界书条目；
- 混合检索补充；
- 历史生成记录；
- 各来源的预算和截断信息。

确认窗口提供“计划”和“实际注入上下文”。取消确认不会修改正式章节。

### 6.3 生成后的监督和维护

确认后，Agent 使用已批准计划生成正文，并执行：

1. 章节监督和必要修复；
2. 正文写入章节树；
3. 摘要生成并绑定章节版本；
4. 世界书章节快照提取和活跃路径重建；
5. 已解决/休眠条目的安全归档；
6. 项目快照保存。

若出现“章节已保存，世界书维护待重试”，表示正文和章节版本已经成功保存，但世界书维护事务失败并回滚。可点击“重试世界书维护”。

### 6.4 Agent 润色和番外

章节树中：

- “Agent 润色”会先分析保真约束和修改范围，再创建新版本；
- “Agent 插入番外”会规划番外位置、类型契约、相关世界书和历史上下文；
- 原章节不会被直接覆盖，新结果以新版本或番外节点保存。

## 7. Agent 写作顾问

### 7.1 顾问考虑哪些信息

顾问的初始事实上下文使用下一章作为任务位置，包含：

- 全局偏好；
- 作者规划；
- 活跃章节路径的前情提要；
- 角色状态、剧情线和伏笔连续性契约；
- 世界书常驻条目、自动命中条目、手动引用和索引；
- 关键词、实体或语义检索结果；
- 历史生成记录。

顾问还保留当前书籍自己的会话历史。为了防止脱离项目胡乱回答，每个书籍相关问题在首次回答前必须调用至少一个章节、项目或世界书工具。

顾问可按需调用：

| 工具能力 | 可读取内容 |
|---|---|
| 章节读取 | 指定活跃章节、指定节点版本、相邻章节范围 |
| 章节搜索 | 正文关键词和章节摘要搜索 |
| 项目摘要 | 总章节数、作者规划、全书摘要和活跃路径 |
| 作者规划 | 主角、背景、写作要求、题材和风格 |
| 项目状态 | 活跃路径、普通摘要和智能摘要 |
| 世界书读取 | 完整世界书、实体搜索和指定实体详情 |
| 上下文报告 | 某一章实际会注入的上下文及来源 |
| 网页搜索 | 仅在启用且问题确实需要外部资料时使用 |

顾问不会自动读取操作系统文件，也不会直接修改正式章节。

### 7.2 顾问会话管理

- 顾问会话按书籍隔离并持续保存。
- 可查看历史、删除单条消息或清空当前书籍顾问会话。
- 长会话超过上下文预算时会压缩历史，保留关键事实、用户决策和未完成事项。

### 7.3 保存构思和加入世界书

“保存构思”将回答存入书籍的加密 Artifact 区，不会改变正文。

“加入世界书”不是直接写入：

1. 世界书管理 Agent 分析回答中的可结构化细节；
2. 生成字段级变更和风险说明；
3. 用户确认每项变更的作用域；
4. 用户批准后才提交正式世界书；
5. 拒绝或取消不会修改正式数据。

## 8. 上下文系统

### 8.1 世界书加载策略

每个世界书实体可设置：

| 策略 | 行为 |
|---|---|
| resident | 每次生成都展开完整条目，适合核心规则和主角 |
| auto | 名称、简介或关键词命中当前任务时展开 |
| manual | 只有用户或 Agent 明确引用实体 ID 时展开 |

未展开的条目仍可作为轻量索引供检索使用。

### 8.2 上下文预览

上下文预览会显示：

- 实际注入的每个区块；
- 来源和命中原因；
- 原始字符数、预算和截断量；
- 常驻、自动、手动和检索结果的区别。

当模型忽略设定时，先检查该设定是否被禁用、是否为 manual、关键词是否匹配，以及预算是否被其他内容占满。

## 9. 世界书

### 9.1 数据结构

世界书主要包含：

- 角色：别名、特征、关系、状态、位置、目标、情绪、行动和知识边界；
- 地点：描述、意义、氛围和关键原文；
- 世界规则：内容、约束、优先级、冲突引用和锁定状态；
- 故事时钟：日期、时段、已流逝时间和故事阶段；
- 时间线：事件、意义、出现次数和关键段落；
- 剧情线：状态、涉及角色、预期回收和最近触达章节；
- 关键设定：核心摘要、完整原文、约束和关键词；
- 伏笔：关联对象、状态、下一步和回收规则；
- 关键对白；
- 来源、事实历史、人工修订和一致性警告。

### 9.2 章节快照和活跃聚合

每个章节版本可以保存独立世界书快照。当前世界书聚合只使用活跃章节路径上的快照，因此切换分支后可以重建对应分支的记忆。

人工修改不会简单写进一次性聚合结果，而是保存为可重放修订。以后重建世界书时，仍会重新应用这些修订。

### 9.3 常用操作

世界书窗口支持：

- 卡片式浏览、筛选和高级文本/JSON 编辑；
- 添加或删除条目；
- 合并角色、审核重复候选、撤销最近合并；
- 查看事实历史和来源章节；
- 标记剧情线或伏笔已解决；
- 锁定核心设定；
- 隐藏低优先级条目；
- 添加作者规划层伏笔；
- 预览某个写作请求的实际世界书注入；
- 查看章节快照、人工修订、迁移和运行诊断。

### 9.4 一致性审计

审计会检查但不限于：

- 缺失或重复实体 ID；
- 关系指向不存在的角色；
- 剧情线引用不存在的角色；
- 设定、伏笔和实体缺少章节来源；
- 角色名或地点名疑似重复；
- 规则冲突和重复；
- 长时间未推进的剧情线和伏笔；
- 世界书为空或章节快照缺失。

审计结果通常用于提示，不应机械地自动删除数据。

### 9.5 重提取和修复

章节树提供：

- 提取当前章节世界书；
- 提取全部活跃路径世界书；
- 按快照重建聚合世界书。

“提取全部”会重新调用模型，成本高于普通重建。仅在快照缺失、结构损坏或需要刷新提取质量时使用。

## 10. 章节树和版本

### 10.1 活跃路径

章节树中的每个节点对应一个具体章节版本。活跃路径决定：

- 阅读和导出的主线；
- 剧情摘要；
- 世界书聚合；
- 下一章的父节点和章节号。

切换分支时，程序会同步剧情记忆和世界书。不要把“章节编号相同”理解为内容相同，版本和父节点同样重要。

### 10.2 节点操作

| 操作 | 结果 |
|---|---|
| 切换分支 | 将选中节点所在路径设为当前阅读/生成路径 |
| 编辑正文 | 打开编辑器并保存新版本 |
| 润色/Agent 润色 | 基于当前内容创建新版本 |
| 重写 | 根据原要求重新生成新版本 |
| 插入番外 | 在选定位置创建番外节点 |
| 重新生成摘要 | 更新该版本摘要并重建全书摘要 |
| 导出章节 | 导出所选节点正文 |
| 删除子树 | 删除节点及所有后代，并重建剩余路径记忆 |

删除子树不可简单撤销，操作前建议创建项目快照。

## 11. 续写已有小说

### 11.1 输入来源

支持：

- 单个 `.txt` 或 `.md` 文件；
- 包含多个 TXT/Markdown 文件的文件夹。

文件夹导入会按文件名和识别出的章节号建立章节。建议使用清晰的命名，例如 `第001章 标题.txt`。

### 11.2 分析和导入流程

1. 选择源文件或目录。
2. 程序按标题和文本结构分段；Agent 模式可使用 Agent 分段器。
3. 在预览窗口确认、合并或调整段落。
4. 创建或选择目标书籍。
5. 按段落保存章节版本。
6. 为每章生成摘要和世界书快照。
7. 从累计世界书生成主角、背景、写作要求和作者规划。

已有目标书非空时，程序会警告，避免误覆盖或混入错误项目。

### 11.3 继续写作

完成分析后可以：

- 让 AI/Agent 建议多个发展方向；
- 选择一个方向后生成；
- 在“我指定剧情”中明确下一章走向；
- 直接生成下一章。

续写上下文会包含源文档尾部、前情摘要、小说设定、连续性契约和相关世界书。

## 12. Markdown 笔记

Markdown 笔记按当前用户保存在 `markdown_workspace` 中。

支持：

- 新建文件夹和 Markdown 文件；
- 左侧目录树管理；
- 编辑与实时 HTML 预览；
- 保存、删除；
- 导出当前文件；
- 将选定文件夹解密导出为普通 Markdown 目录。

切换工作区或退出前，如果当前笔记尚未保存，程序会提示保存、放弃或取消。

导出后的 Markdown 是普通文件，不受应用加密保护。

## 13. 项目快照

项目快照与章节版本解决不同问题：

- 章节版本记录某一章的多个文本版本和分支关系；
- 项目快照原子保存整本书的章节、摘要、世界书、元数据和内部配置。

快照来源包括：

- 章节生成后的自动快照；
- 定时快照，默认间隔由设置控制；
- 高风险维护操作前的保护快照；
- 用户手动创建的项目版本。

恢复项目版本前会先创建备份，并在校验通过后保留恢复结果。

## 14. 导出和数据管理

### 14.1 内容导出

小说和对话支持 TXT、Markdown、HTML、DOCX。章节树中的单节点导出通常提供 TXT/Markdown。

导出只复制内容，不会改变书籍活跃路径或版本。

### 14.2 用户数据包

“设置中心 → 数据管理”支持：

- 导出当前用户数据包；
- 导入用户数据包；
- 清空当前用户数据。

数据包用于迁移和备份。导入后建议重启程序，以确保所有缓存和管理器重新加载。

### 14.3 旧版数据迁移

程序检测到旧版根目录中的明文书架或对话时，会询问是否迁移到当前用户目录并加密保存。迁移完成前不要手工删除旧数据。

## 15. Token 日志

Token 日志记录模型调用的时间、操作类型、模式、模型、输入输出 Token、耗时和内容预览。

用途包括：

- 判断哪个环节消耗最多；
- 检查某次调用是否使用预期模型；
- 排查长上下文和输出截断；
- 对比经典和 Agent 模式成本。

Token 数由供应商响应提供时最准确；部分兼容接口可能不返回完整统计。

## 16. 设置中心

### 16.1 API 与模型

配置文字/图片 API，保存后立即应用。API Key 加密保存在当前用户目录。

### 16.2 生成参数

管理参数预设、默认模型、收藏模型和自定义模型。

### 16.3 账号安全

修改密码需要输入旧密码。修改成功后，应用会重新加密当前用户目录下的加密文件。过程中不要强制退出程序。

### 16.4 数据管理

提供数据包导入导出和清空用户数据。清空不可恢复。

### 16.5 外观

可选择主题等界面设置。部分对话框会根据屏幕大小自动调整。

### 16.6 Agent

可切换经典/Agent 写作模式，配置 Skills、运行时、检索、Embedding 和网页搜索。

网页搜索工具只返回受控的标题、链接、摘要和查询时间；网页内容被视为不可信数据，不能覆盖 Agent 的系统规则。

## 17. 本地数据和安全

### 17.1 存储位置

数据根目录为项目下的 `users/`：

```text
users/
├── users.json                 账号盐值、认证摘要和随机目录映射
└── <随机目录 ID>/
    ├── bookshelf/             小说项目
    ├── conversations/         对话记录
    ├── markdown_workspace/    Markdown 笔记
    ├── config.enc             API 配置
    ├── settings.json.enc      设置
    └── ...                    角色簿、Token 日志等
```

书籍内部通常包含元数据、章节文件、摘要、世界书和 `.deepseekass/` 内部目录。内部目录保存 manifest、上下文策略、Agent 记录和项目快照。

### 17.2 安全规则

- 不要把 `users/` 提交到公共代码仓库。
- 不要共享包含 API Key 的用户数据包。
- 不要直接编辑 `.enc` 文件。
- 重要项目应同时保留应用数据包和导出的正文副本。
- 应用加密不能防止已登录系统账号读取正在运行的程序内存或导出文件。

## 18. 常见问题

### 密码错误为什么无法找回？

密码参与派生加密密钥，程序没有可用于恢复原密钥的后门。只能使用正确密码，或删除账号数据重新开始。

### 密码错误弹窗为什么会连续出现？

当前版本已经对登录操作增加防重复提交。若仍发生，请确认运行的是最新源码并重启程序。

### 笔记区显示问号怎么办？

当前源码已恢复 Markdown 工作区中文文案。旧程序包需要重新构建或更新后启动。

### “章节已保存，世界书维护待重试”是什么意思？

章节正文、版本和通常的项目快照已经保存；世界书维护事务失败并回滚。点击“重试世界书维护”，或在章节树中重提当前章节世界书。

### 为什么出现 `Project snapshot saved.`？

表示生成完成后已创建整书项目快照。这是成功信息，不是错误。

### Agent 顾问会读整本正文吗？

不会默认无差别读取整本正文。它先接收预算内摘要、连续性和世界书上下文，再根据问题调用章节读取或搜索工具。

### 顾问回答能自动改世界书吗？

不能。保存构思只写入隔离 Artifact；加入世界书需要生成变更计划、确认作用域并批准。

### 为什么设定没有进入生成上下文？

检查世界书加载策略、实体是否被隐藏、关键词是否命中、手动实体是否选择，以及上下文预览中的截断情况。

### 切换章节分支后为什么摘要变了？

摘要和世界书按活跃路径重建。不同版本或父节点属于不同剧情路径。

### 世界书出现错误条目怎么办？

可编辑或删除条目、锁定正确设定、合并重复实体，或重提当前章节快照。大规模重提前先创建项目快照。

### 生成字数为什么少于目标？

目标字数只是提示约束，仍受模型能力、服务端 max_tokens 和上下文长度限制。提高输出上限或拆分章节通常更可靠。

### LangGraph 或 LlamaIndex 配置失败还能用吗？

可以使用内置 Agent 运行时和基础关键词检索。可选后端不应成为基础写作功能的必需条件。

### 如何报告问题？

保留以下信息：

- 错误提示全文；
- 使用的工作区和经典/Agent 模式；
- 操作发生在生成、摘要、世界书还是快照阶段；
- Token 日志中的操作名；
- 可复现步骤。

不要公开 API Key、密码、加密数据或包含隐私的小说正文。
