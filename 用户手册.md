# 用户手册

> 本文档对应应用版本：2026-05

## 一、启动与登录

首次运行 `python gui_main.py` 后会弹出登录窗口：

- **已有账号**：输入用户名和密码登录
- **注册新账号**：勾选"注册新账号"，输入用户名 + 两次密码确认

登录后，程序会自动验证 API Key 有效性，验证失败会弹出提示。

密码用于加密*所有*本地数据（书架、对话、配置），请妥善保管。

## 二、三种模式

| 模式 | 说明 |
|------|------|
| 🎭 角色扮演 | 让 AI 扮演某个角色和你对话（比如骑士、猫娘、名将、反派） |
| 📚 小说写作 | 让 AI 帮你从零写小说，书架管理、章节续写、自动记剧情 |
| 📄 续写小说 | 给你已有的 .txt/.md 文章，AI 分析后接着往下写后续章节 |

启动后在左上角"聊天模式"下拉框切换，切换后左侧面板会显示对应模式的专属设定项。

---

## 三、界面布局

窗口分左右两栏。左侧为控制面板（可滚动），右侧为对话区。

**左侧面板（自上而下）：**

| 区域 | 内容 |
|------|------|
| 📌 聊天模式 | 三种模式下拉切换 |
| 🧠 模型选择 | 选模型 |
| ⚙️ 生成参数 | 参数预设方案 + 温度/top_p/freq_p/max_tokens 微调 |
| 🛑 停止生成 | 流式输出中显示红色停止按钮，点击立即终止 |
| 模式专属面板 | 根据当前模式显示角色扮演 / 小说写作 / 续写小说面板 |
| 📋 状态 | 当前模式与模型 |

右侧对话区分上下两部分：
- 上方：Markdown 渲染的聊天记录，实时显示流式输出和字符计数
- 下方：输入框（Ctrl+Enter 发送）+ 发送按钮，生成中显示停止按钮

## 四、全局通用

### 4.1 模型选择

| 模型 | 特点 |
|------|------|
| `deepseek-v4-flash` | 快速，日常使用 |
| `deepseek-v4-pro` | 更强，复杂任务或长文生成 |

### 4.2 生成参数

| 参数 | 作用 | 范围 |
|------|------|------|
| 预设方案 | 保守 / 中庸 / 狂野 一键切换，也可手动微调；手动调节任意滑块自动切换为"自定义" | — |
| 🌡️ 温度 | 控制创造力，越高越天马行空 | 0 ~ 2 |
| 🎯 top_p | 采样阈值，一般保持默认 | 0 ~ 1 |
| 🔄 freq_p | 频率惩罚，越高越不重复用词 | -2 ~ 2 |
| 📏 max_tk | AI 单次最大输出 token 数，写长文时调大 | 1 ~ 300000 |

**预设方案说明：**

| 预设 | 温度 | Top P | 频率惩罚 | 适用场景 |
|------|------|-------|---------|---------|
| 保守 | 0.30 | 0.50 | +0.30 | 事实性内容、严格续写 |
| 中庸 | 0.70 | 0.90 | 0.00 | 日常聊天、常规写作 |
| 狂野 | 0.90 | 1.00 | +0.10 | 创意脑暴、发散情节 |
| 自定义 | — | — | — | 手动微调任意参数后自动变为自定义 |

### 4.3 全局偏好提示词

在任意模式下，点击左侧面板的"📝 全局偏好提示词"按钮，可以编辑一段持续生效的个性化偏好文本。这段内容会自动注入到*每一次* API 调用中——包括章节生成、续写、摘要、世界书提取、方向建议和字数补充。

典型用途：
- "偏好优美细腻的文风，多用比喻和通感"
- "对话要多，心理描写要深"
- "避免套路化剧情，追求反套路"

### 4.4 停止生成

流式输出过程中，输入框旁边会出现红色 **■ 停止** 按钮。点击后立即终止当前 AI 生成，已输出的内容保留在对话区。

切换到此模式后，左侧显示角色扮演设定面板。

| 控件 | 说明 |

| 💾 保存当前对话 | 把当前对话存为加密文件，以后可重新打开 |
| 📂 加载 | 打开选中的对话，自动切换到当时的模式和模型 |
| 🗑️ 删除 | 删除选中的对话记录 |
| 📤 导出对话 | 导出为 TXT / Markdown / HTML / Word |

## 六、📚 小说写作模式

### 6.1 书架管理

| 控件 | 说明 |
|------|------|
| 📖 书架（下拉框） | 选择已创建的小说 |
| ➕ 新建 | 创建新小说 |
| ✏️ 重命名 | 修改当前小说的名称 |
| 🗑️ 删除 | 删除整本小说（含所有章节，不可恢复） |
| ✏️ 标题 | 当前小说标题，改了自动保存 |

### 6.2 章节控制

| 控件 | 说明 |
|------|------|
| 📝 章节标题 | 下一章标题，如"少年踏上征途" |
| ℹ️ 章节信息 | 显示已有章节数和下一章编号（自动更新） |
| ✅ 章节续写模式 | 勾选后，发送消息 = 生成下一章 | 不勾 = 正常聊天 |
| ⚙️ 章节管理 | 打开章节版本管理对话框（见后文） |

### 6.3 小说设定

| 控件 | 说明 |
|------|------|
| 👤 主角设定 | 主角背景/性格/外貌，改了自动保存 |
| 🌍 世界观/背景 | 时代背景、核心设定，改了自动保存 |
| ✍️ 写作要求 | 本章特别要求，改了自动保存 |
| 📝 本章情节（可选） | 你想写的关键情节，AI 以此为基础扩展，留空则 AI 自由创作 |
| 🔢 字数 | 本章目标字数，默认 10000 |
| 💾 / 📂 保存/加载设定 | 手动保存或加载当前小说的设定文件 |

### 6.4 生成流程

点击 **🚀 生成下一章** 后，AI 自动执行以下流程：

1. 📦 收集主角设定、世界观、写作要求、全局偏好提示词
2. 📖 读取前情提要（短篇全读，长篇自动压缩早期章节，保留最近 3 章详细内容）
3. 📕 读取世界书（已提取的角色、地点、规则、剧情线、伏笔）
4. 🤖 AI 生成章节正文
5. 💾 自动保存到书架（生成记录同时保存至 `.generation_history/`）
6. 📋 提炼本章摘要并追加到剧情摘要
7. 📏 字数不足目标 80% 时自动补写（调用 AI 扩充内容，存为新版本）
8. 🔄 提取本章新角色/新地点/新线索，更新世界书

### 6.5 章节版本管理

点击 **⚙️ 章节管理** 打开版本管理窗口。

| 操作 | 说明 |
|------|------|
| 👁️ 预览 | 查看某版本的完整内容（只读对话框） |
| ⭐ 设为活跃（计入剧情） | 把某版本设为正式版（只有活跃版参与剧情摘要），可选是否重建摘要 |
| 🔁 重新生成 | 用当前设定重写某章，基于原版要求和历史参数，存为新版本（v+1），不覆盖旧版 |
| 🗑️ 删除此版本 | 删除某版本（不能删除活跃版本） |

**重新生成（🔁）** 会从该章的历史记录中读取原始的需求和剧情方向，结合当前世界书和智能摘要，让 AI 以原版风格参考重新创作。生成完成后自动更新世界书。

### 6.6 辅助工具

| 按钮 | 说明 |
|------|------|
| 📖 世界书 | 查看/编辑 AI 自动提取的世界书数据——角色（含别名、关系、重要性）、地点、规则、时间线、剧情线、设定与伏笔 |
| 📄 分段摘要 | 选择一个 .txt/.md 文件，AI 自动按语义分段并为每段生成摘要 |
| 📂 批量导入章节 | 选择一个文件夹（包含编号的 .txt 文件），批量导入为小说章节（按文件名排序逐章处理，自动提取世界书和生成设定） |

### 6.7 导出

| 控件 | 说明 |
|------|------|
| 导出格式（下拉框） | TXT / Markdown / HTML / Word（DOCX） |
| 📄 导出当前章节 | 导出最新一章 |
| 📚 导出全书 | 导出所有章节，Word 格式会包含封面页、设定页、目录和分页符 |

## 七、📄 续写小说模式

用于给已有文章续写后续章节。核心流程：**选择源文档 → 预览分段（可选 AI 重新分段）→ 分析提取设定（可选）→ 续写操作**。

### 7.1 源文档

| 控件 | 说明 |
|------|------|
| 📄 源文档 → 浏览 | 选择一个 .txt / .md 文件作为续写来源 |
| 📁 或指定文件夹 → 浏览 | 选一个文件夹，程序读取其中所有 .txt/.md/.html 文件（按文件名排序拼接） |

选择文件后弹出**分段预览对话框**：
- 左侧：AI 自动划分的段落列表
- 右侧：当前选中段落的完整内容
- 🔄 AI 重新分段：不满意自动划分结果时，点击让 AI 重新按语义切分
- 文件夹模式会列出所有文件，可勾选需要包含的文件

### 7.2 分析源文档

**🔍 分析源文档并导入设定：**
先让 AI 读取全文，自动执行以下步骤：
1. **语义分段**：按话题转折划分逻辑段落
2. **结构化提取**：提取角色、地点、规则、剧情线
3. **生成小说设定**：世界观背景、主角描述、写作要求
4. **创建书架 + 保存世界书 + 加载设定到面板**
5. 弹出**分析结果总览窗口**（包含生成设定、角色、地点、规则、剧情线、事件等选项卡）

**⚡ 直接续写：** 跳过分析，直接用原文件和手动填写的设定续写。

### 7.3 书架与章节管理

分析源文档后自动创建小说，与小说写作模式共用同一套书架系统。

| 控件 | 说明 |
|------|------|
| 📖 书架（下拉） + 新建/删除/重命名 | 管理续写项目 |
| 📝 章节标题 | 本章标题 |
| ℹ️ 章节信息 | 显示已有章节数 |
| ✅ 章节续写模式 | 勾选后发送即生成下一章 |
| ⚙️ 章节管理 | 章节版本管理对话框（含预览/重新生成/版本切换） |

### 7.4 小说设定

| 控件 | 说明 |
|------|------|
| 👤 主角设定 | 改了自动保存 |
| 🌍 世界观/背景 | 改了自动保存 |
| ✍️ 写作要求 | 改了自动保存 |
| 📝 全局偏好提示词 | 与通用全局偏好共用 |
| 💾 / 📂 保存/加载设定 | 手动操作 |

### 7.5 续写操作

| 控件 | 说明 |
|------|------|
| ✍️ 续写要求 | 对续写提风格/视角/节奏等要求 |
| 🔢 字数 | 目标字数，默认 10000 |
| 📝 续写剧情（可选） | 可在此填写你想写的剧情走向、关键事件 |

三个生成入口（位于左侧面板「续写剧情」下方）：

| 按钮 | 说明 |
|------|------|
| 🎲 AI 建议发展方向 | AI 根据已分析的故事设定和活跃剧情线，给出 3—5 个下一章的发展方向建议（含标题、核心冲突、大致情节走向），选择后自动续写 |
| 📝 我指定剧情 | 将你在"续写剧情"框中的内容，结合已分析的剧情上下文传给 AI 续写 |
| 🚀 生成下一章 | 直接执行续写，自动注入前文剧情和设定 |

### 7.6 导出

与小说写作模式相同：TXT / Markdown / HTML / Word（DOCX）格式，支持单章和全书导出。

## 八、自动行为

1. 💾 **自动保存**：修改主角/世界观/写作要求时自动保存到 `meta.json`
2. 📏 **自动补写**：章节字数不足目标 80% 时，自动调用 API 补写内容，存为新版本（不覆盖原版）
3. 📖 **自动更新世界书**：每生成一章，自动提取新角色、新地点、新规则、新剧情线（含别名检测与合并）
4. 📋 **章节版本管理**：每章可保留多个版本，只有活跃版本参与剧情摘要；切换活跃版本时可选择重建摘要
5. 📄 **智能摘要压缩**：长篇小说（>5 章）自动压缩早期章节摘要为 ~300 字概览，保留最近 3 章详细信息
6. 📜 **生成历史记录**：每次生成/续写都保存详细 JSON 记录（含提示词、参数、内容预览），用于重新生成时还原原始需求
7. 🔍 **API Key 验证**：启动时自动验证 API Key 有效性

## 九、数据存储与安全

### 9.1 加密机制

所有本地数据均使用**Fernet 加密**（基于登录密码通过 PBKDF2-SHA256 派生的密钥），以 `.enc` 加密文件形式存储。没有密码无法读取任何数据。

### 9.2 存储结构

```
users/
└── <用户 UUID>/
    ├── config.enc                # API Key / Base URL（加密）
    ├── user_prefs.enc            # 全局偏好提示词（加密）
    ├── bookshelf/
    │   └── <小说名>/
    │       ├── meta.json.enc     # 元信息（设定/章节版本/字数/创建时间）
    │       ├── plot_summary.txt.enc  # 剧情摘要
    │       ├── world_bible.json.enc  # 世界书（角色/地点/规则/剧情线/伏笔）
    │       ├── 第1章.txt.enc        # 章节正文（含版本历史）
    │       └── .generation_history/ # 每次生成的详细日志
    └── conversations/
        └── <对话名>.json.enc     # 角色扮演对话历史
```

## 十、常见问题

**Q：忘记密码了怎么办？**
A：加密密钥基于密码派生，忘记密码无法恢复数据。请务必记住密码。

**Q：如何更换 API Key？**
A：登录后在设置区域修改 API Key 和 Base URL，修改后会自动加密保存。

**Q：生成速度慢怎么办？**
A：选择 `deepseek-v4-flash` 模型可显著提升速度；降低 max_tokens 也能减少等待时间。

**Q：生成被中断了，已生成的内容还在吗？**
A：点击停止按钮后，已流式输出的内容保留在对话区，但尚未保存到书架。需要手动复制保存或重新生成。
