# 用户手册

## 一、三种模式

| 模式 | 说明 |
|------|------|
| 🎭 角色扮演 | 让 AI 扮演某个角色和你对话（比如骑士、猫娘、名将、反派） |
| 📚 小说写作 | 让 AI 帮你从零写小说，书架管理、章节续写、自动记剧情 |
| 📄 续写小说 | 给你已有的 .txt/.md 文章，AI 分析后接着往下写后续章节 |

启动后在左上角"聊天模式"下拉框切换，切换后左侧面板会显示对应模式的专属设定项。

---

## 二、界面布局

窗口分左右两栏。左侧为控制面板（可滚动），右侧为对话区。

**左侧面板（自上而下）：**

| 区域 | 内容 |
|------|------|
| 📌 聊天模式 | 三种模式下拉切换 |
| 🧠 模型选择 | 选模型 |
| ⚙️ 生成参数 | 参数预设方案 + 温度/top_p/freq_p/max_tokens 微调 |
| 操作 | 清除对话（仅角色扮演模式显示） |
| 💬 对话历史 | 保存/加载/删除/导出对话（仅角色扮演模式） |
| 📋 状态 | 当前模式与模型 |
| 模式专属面板 | 根据当前模式显示角色扮演 / 小说写作 / 续写小说面板 |

**右侧对话区：**
- 上方：Markdown 渲染的聊天记录
- 下方：输入框（Ctrl+Enter 发送）+ 发送按钮

## 三、全局通用

### 模型选择

| 模型 | 特点 |
|------|------|
| `deepseek-v4-flash` | 快速，日常使用 |
| `deepseek-v4-pro` | 更强，复杂任务或长文生成 |

### 生成参数

| 参数 | 作用 | 范围 |
|------|------|------|
| 预设方案 | 保守 / 中庸 / 狂野 一键切换，也可手动微调 | — |
| 🌡️ 温度 | 控制创造力，越高越天马行空 | 0 ~ 2 |
| 🎯 top_p | 采样阈值，一般保持默认 | 0 ~ 1 |
| 🔄 freq_p | 频率惩罚，越高越不重复用词 | -2 ~ 2 |
| 📏 max_tk | AI 单次最大输出 token 数，写长文时调大 | 1 ~ 300000 |

## 四、🎭 角色扮演模式

切换到此模式后，左侧显示角色扮演设定面板。

| 控件 | 说明 |
|------|------|
| 👤 角色描述 | 要 AI 扮演谁，如"一位傲娇的中世纪骑士" |
| 🌍 故事背景 | 故事发生的世界，如"中世纪欧洲，有魔法" |
| 💬 回复方式（单选） | 角色回答（第一人称"我"）或 旁白描述（第三人称"他/她"） |
| ✅ 应用设定 | 将上述内容写入 AI 指令，保留聊天记录不清除 |

**对话历史：**
| 按钮 | 作用 |
|------|------|
| 💾 保存当前对话 | 把当前对话存为 JSON，以后可重新打开 |
| 📂 加载 | 打开选中的对话，自动切换到当时的模式和模型 |
| 🗑️ 删除 | 删除选中的对话记录 |
| 📤 导出对话 | 导出为 TXT / Markdown / HTML / Word |

## 五、📚 小说写作模式

### 5.1 书架管理

| 控件 | 说明 |
|------|------|
| 📖 书架（下拉框） | 选择已创建的小说 |
| ➕ 新建 | 创建新小说 |
| 🗑️ 删除 | 删除整本小说（含所有章节，不可恢复） |
| ✏️ 标题 | 当前小说标题，改了自动保存 |

### 5.2 章节控制

| 控件 | 说明 |
|------|------|
| 📝 章节标题 | 下一章标题，如"少年踏上征途" |
| ℹ️ 章节信息 | 显示已有章节数和下一章编号（自动更新） |
| ✅ 章节续写模式 | 勾选后，发送消息 = 生成下一章 | 不勾 = 正常聊天 |

### 5.3 小说设定

| 控件 | 说明 |
|------|------|
| 👤 主角设定 | 主角背景/性格/外貌，改了自动保存 |
| 🌍 世界观/背景 | 时代背景、核心设定，改了自动保存 |
| ✍️ 写作要求 | 本章特别要求，改了自动保存 |
| 📝 本章情节（可选） | 本章你想写的关键情节，AI 以此为基础扩展，留空则 AI 自由创作 |
| 🔢 字数 | 本章目标字数，默认 10000 |

**💾 保存/加载设定按钮：** 手动保存或加载当前小说的设定文件。

### 5.4 生成流程

点击 **🚀 生成下一章** 后，AI 自动执行以下流程：

1. 📦 收集主角设定、世界观、写作要求
2. 📖 读取前情提要（短篇全读，长篇自动压缩早期章节）
3. 📕 读取世界书（已提取的角色、地点、规则、剧情线）
4. 🤖 AI 生成章节正文
5. 💾 自动保存到书架
6. 📋 提炼本章摘要并追加到剧情摘要
7. 📏 字数不足目标 80% 时自动补写
8. 🔄 提取本章新角色/新地点/新线索，更新世界书

### 5.5 章节版本管理

点击 **⚙ 章节管理** 打开版本管理窗口。

| 操作 | 说明 |
|------|------|
| 👁️ 预览 | 查看某版本的完整内容 |
| ⭐ 设为活跃（计入剧情） | 把某版本设为正式版（只有活跃版参与剧情摘要），可选是否重建摘要 |
| 🔁 重新生成 | 用当前设定重写某章，存为新版本（v+1），不覆盖旧版 |
| 🗑️ 删除此版本 | 删除某版本（不能删除活跃版本） |

### 5.6 辅助工具

| 按钮 | 说明 |
|------|------|
| 📖 世界书 | 查看/编辑 AI 自动提取的角色、地点、规则、剧情线、伏笔 |
| 📄 分段摘要 | 选择一个 .txt/.md 文件，AI 自动按语义分段并为每段生成摘要 |

### 5.7 导出

| 控件 | 说明 |
|------|------|
| 导出格式（下拉框） | 选择 TXT / Markdown / HTML / Word |
| 📄 导出当前章节 | 导出最新一章 |
| 📚 导出全书 | 导出所有章节 |

## 六、📄 续写小说模式

用于给已有文章续写后续章节。核心流程：**选择源文档 → 分析提取设定（可选）→ 续写操作**。

### 6.1 源文档

| 控件 | 说明 |
|------|------|
| 📄 源文档 → 浏览 | 选择一个 .txt / .md 文件作为续写来源 |
| 📁 或指定文件夹 → 浏览 | 选一个文件夹，程序读取其中所有 .txt/.md/.html 文件（按文件名排序拼接） |

**🔍 分析源文档并导入设定：**
先让 AI 读取全文，自动做以下事情：
1. 语义分段：按话题转折划分逻辑段落
2. 结构化提取：提取角色、地点、规则、剧情线
3. 生成小说设定：世界观背景、主角描述、写作要求
4. 创建书架 + 保存世界书 + 加载设定到面板
5. 弹出分析结果总览窗口

**⚡ 直接续写：** 跳过分析，直接用原文件和手动填写的设定续写。

### 6.2 书架与章节管理

与小说写作模式相同的书架系统。分析源文档后自动创建小说并填入设定。

| 控件 | 说明 |
|------|------|
| 📖 书架（下拉） + 新建/删除 | 管理续写项目 |
| 📝 章节标题 | 本章标题 |
| ℹ️ 章节信息 | 显示已有章节数 |
| ✅ 章节续写模式 | 勾选后发送即生成下一章 |

### 6.3 小说设定

| 控件 | 说明 |
|------|------|
| 👤 主角设定 | 改了自动保存 |
| 🌍 世界观/背景 | 改了自动保存 |
| ✍️ 写作要求 | 改了自动保存 |
| 💾/📂 保存/加载设定 | 手动操作 |

### 6.4 续写操作

| 控件 | 说明 |
|------|------|
| ✍️ 续写要求 | 对续写提风格/视角/节奏等要求 |
| 🔢 字数 | 目标字数，默认 10000 |
| 📝 续写剧情（可选） | 可在此填写你想写的剧情走向、关键事件 |

**🎲 AI 建议发展方向**
点击后 AI 根据已分析的故事设定和活跃剧情线，给出 3—5 个下一章的发展方向建议（含标题、核心冲突、大致情节走向），选择后自动续写。

**📝 我指定剧情**
将你在"续写剧情"框中填写的内容，结合已分析的剧情上下文，传给 AI 续写。

> 这两个按钮位于左侧面板「续写剧情」下方，分析过源文档后随时可用，无需重新打开弹窗。

**🚀 生成下一章**
直接执行续写，自动注入前文剧情和设定。

### 6.5 导出

| 控件 | 说明 |
|------|------|
| 导出格式（下拉框） | TXT / Markdown / HTML / Word |
| 📄 导出当前章节 | 导出最新一章 |
| 📚 导出全书 | 导出所有章节 |

## 七、自动行为

1. 💾 **自动保存**：修改主角/世界观/写作要求时自动保存到 `meta.json`
2. 📏 **自动补写**：章节字数不足目标 80% 时，自动调用 API 补写内容
3. 📖 **自动更新世界书**：每生成一章，自动提取新角色、新地点、新规则、新剧情线
4. 📋 **章节版本管理**：每章可保留多个版本，只有活跃版本参与剧情摘要
5. 📄 **智能摘要压缩**：长篇小说自动压缩早期章节摘要，保留近期章节详细信息

## 八、数据存储

所有数据以 JSON 文件形式存储在项目目录下：

```
bookshelf/
└── <小说名>/
    ├── meta.json               # 元信息（设定/章节版本/字数/创建时间）
    ├── plot_summary.txt        # 剧情摘要
    ├── world_bible.json        # 世界书（角色/地点/规则/剧情线/伏笔）
    ├── 第1章.txt / 第2章.txt   # 章节正文（含版本历史）
    └── .generation_history/    # 每次生成的详细日志

conversations/
└── <对话名>.json               # 角色扮演对话历史
```
